Пояснительная записка

Запуск генератора отчетов производится через terminal/cmd командой:

java generator settings.xml source-data.tsv example-report.txt


XML файл с настройками и дата-файл должны находится в той же дериктории, что и
исполняемый файл. Отчет будет сгенерирован там же.


Генератор поддерживает изменение геометрических параметров таблицы.


Генератор не поддерживает:
   -изменение количества колонок в таблице
   -изменение структуры XML файла с настройками
   -изменение структуры файла с базой данных

Этапы работы:

1) Загрузка параметров из XML файла, инициализация.
   Чтение файла производится с помощью библиотеки javax.xml.parsers.*
   Далее переменные класса инициализируются загруженными параметрами, создается
   шаблон, по которому собираются строки таблицы (String.format).

2) Чтение файла с данными и генерация отчета
   В main() производится вызов метода getPages(), который возвращает готовый отчет в виде
   списка строк. Создание отчета реализовано в виде построчной сборки страниц из
   элементов: 

   header     - заголовок таблицы (номер | дата | фио)
   borderLine - разделитель строк "----"
   borderChar - разделитель страниц "~"
   dataLine   - строка с данными

   В случае превышения количества суммарных строк на странице в процессе сборки, производится
   досрочное "закрытие" текущего листа символом "~", текущая строка с данными переносится
   целиком на новый лист. 

   Создание строки с данными производится либо простым форматированием полученной
   строки из файла, либо разбивается на подстроки в случае невозможности уместить данные 
   в одну ячейку по ширине: строка ячейки сначала разделяется на слова, затем разделяются те
   слова, которые сами по себе не помещаются в ячейку по ширине, в конце собирается финальный
   список подстрок ячейки в соответствии с заданной шириной столбца.

3) Запись в файл
